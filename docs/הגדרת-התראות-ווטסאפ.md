# הגדרת התראות WhatsApp (Twilio)

היישום שולח התראות WhatsApp אוטומטיות (הודעה חדשה בצ'אט משימה, שיוך משימה, אישור
קבלה) — **כאשר מוגדר ספק**. ללא הגדרה, ההתראות עדיין עובדות ב**פעמון בתוך היישום
וב-מייל**; ה-WhatsApp פשוט לא יישלח.

> **למה צריך ספק?** WhatsApp לא מאפשר לשלוח הודעות אוטומטיות ממספר רגיל. נדרש
> "שולח עסקי" מאושר. הדרך החינמית והיציבה ביותר לבדיקות היא ה-**Sandbox של Twilio**.

---

## אפשרות א' — Twilio Sandbox (חינמי, לבדיקות / צוות קטן)

1. פתח/י חשבון חינמי ב-[twilio.com](https://www.twilio.com/try-twilio).
2. בקונסול: **Messaging → Try it out → Send a WhatsApp message**. תקבל/י:
   - מספר ה-Sandbox (לדוגמה `+1 415 523 8886`)
   - מילת הצטרפות (לדוגמה `join clever-fox`)
3. **כל חבר צוות** שאמור לקבל התראות שולח פעם אחת מה-WhatsApp שלו את מילת
   ההצטרפות (`join clever-fox`) למספר ה-Sandbox. (חד-פעמי.)
4. העתק/י מהקונסול: **Account SID** ו-**Auth Token**.
5. ב-Vercel → Project → **Settings → Environment Variables**, הוסף/י:
   ```
   TWILIO_ACCOUNT_SID   = AC********************************
   TWILIO_AUTH_TOKEN    = ********************************
   TWILIO_WHATSAPP_FROM = whatsapp:+14155238886
   ```
6. **Redeploy** (Deployments → ⋯ → Redeploy). זהו — התראות WhatsApp פעילות.

> מגבלות ה-Sandbox: רק נמענים שהצטרפו, ויש להגיב בתוך 24 שעות אם הם כותבים. מספיק
> בהחלט לצוות פנימי.

## אפשרות ב' — מספר WhatsApp עסקי מאושר (לפרודקשן מלא)

לשליחה לכל מספר ללא הצטרפות מראש נדרש מספר WhatsApp Business מאושר + תבניות הודעה
מאושרות (Twilio Senders, או Meta Cloud API). לאחר האישור — אותם משתני סביבה
(`TWILIO_*`), רק עם המספר המאושר ב-`TWILIO_WHATSAPP_FROM`.

### חלופה: Meta WhatsApp Cloud API
```
WHATSAPP_PHONE_NUMBER_ID = ...
WHATSAPP_ACCESS_TOKEN    = ...
```

## בדיקה
לאחר ה-Redeploy: צור משימה ושייך חבר צוות (שהצטרף ל-Sandbox), או כתוב הודעה בצ'אט
המשימה — חבר הצוות אמור לקבל הודעת WhatsApp (וגם פעמון + מייל).

המספרים נשלפים משדה הטלפון של המשתמש (טל' ישראלי `05X` מומר אוטומטית ל-`+9725X`).

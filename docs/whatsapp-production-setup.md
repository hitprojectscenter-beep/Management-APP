# WhatsApp בפרודקשן — מסירה לכולם בלי "join" (PMO++)

היום המערכת שולחת דרך **Twilio Sandbox**, שמוסר הודעה רק למי שביצע פעם אחת
`join elephant-bread`. כדי שכל משתמש יקבל WhatsApp **בלי** שלב ההצטרפות, צריך
**WhatsApp Sender מאושר** + **תבנית (Template) מאושרת**. זה חוק של WhatsApp/Meta,
לא מגבלה של המערכת.

הקוד כבר **מוכן**: ברגע שיהיה לך Sender מאושר + תבנית, מגדירים 2–3 משתני סביבה
ב-Vercel — **בלי שינוי קוד**.

---

## מה אני (הקוד) כבר עשיתי
`lib/notify/transport.ts` תומך עכשיו בשני מצבים ב-Twilio, לפי משתני סביבה:
- **פרודקשן:** אם מוגדר `TWILIO_WHATSAPP_CONTENT_SID` — ההודעה נשלחת כמשתנה `{{1}}`
  של תבנית מאושרת (הדרך היחידה לשלוח יזומה מחוץ לחלון 24 שעות).
- **סנדבוקס/חלון 24ש':** בלי Content SID — נשלח טקסט חופשי (לבדיקות בלבד).
המערכת מנקה אוטומטית שורות חדשות מהמשתנה (Meta אוסר תווי newline בתוך משתני תבנית).

---

## מה עליך לעשות (חד-פעמי; דורש חשבון Twilio + Meta)

### שלב 1 — שדרוג חשבון Twilio מ-Trial
בקונסולת Twilio → שדרג/הוסף אמצעי תשלום (Trial לא יכול לשלוח יזומות לפרודקשן).

### שלב 2 — רישום WhatsApp Sender
Twilio Console → **Messaging → Senders → WhatsApp senders → Create new sender**.
- זה יפתח **הרשמה משובצת של Meta** (Embedded Signup): מתחברים ל-**Meta Business
  account** (או פותחים חדש) ומאמתים את העסק ("המרכז למיפוי ישראל").
- **מספר טלפון ל-WhatsApp:** חייב להיות מספר ש**אינו** בשימוש ב-WhatsApp אישי
  (מספר ארגוני חדש / קו יעודי). Twilio ישלח קוד אימות למספר.
- **אימות עסקי (Business Verification) של Meta** עלול לקחת **בין כמה שעות למספר
  ימים**, ולעיתים נדרשים מסמכי עסק. זה הצוואר-בקבוק היחיד בלוח הזמנים.

### שלב 3 — יצירת תבנית מאושרת (Content Template)
Twilio Console → **Content Template Builder → Create new**:
- **Name:** `pmo_notification`
- **Category:** `Utility` (התראה תפעולית — לא שיווק)
- **Language:** `Hebrew (he)`
- **Body** (הדבק בדיוק כך, כולל `{{1}}`):
  ```
  עדכון ממערכת ניהול המשימות PMO++ של המרכז למיפוי ישראל:

  {{1}}
  ```
- **Sample value ל-{{1}}** (למילוי בטופס האישור):
  `משימה חדשה שויכה אליך — הכנת דוח רבעוני, לביצוע עד 15/07. פרטים ביישום.`
- שלח **Submit for WhatsApp approval**. אישור Utility בדרך כלל מהיר (דקות עד שעות).

> אם התבנית נדחית כ"כללית מדי": אפש‏ר לפצל לתבניות ייעודיות (הזמנה / התראת משימה)
> עם משתנים נפרדים (שם, קישור, סיסמה). תגיד לי ואבנה את הקוד שמזרים משתנים מובנים.

### שלב 4 — הגדרת משתני הסביבה ב-Vercel
Vercel → Project **management-app** → Settings → Environment Variables (Production):
| משתנה | ערך |
|---|---|
| `TWILIO_ACCOUNT_SID` | (כפי שכבר מוגדר) |
| `TWILIO_AUTH_TOKEN` | (כפי שכבר מוגדר) |
| `TWILIO_WHATSAPP_FROM` | `whatsapp:+<המספר המאושר שלך>` |
| `TWILIO_WHATSAPP_CONTENT_SID` | ה-SID של התבנית (מתחיל ב-`HX…`) |

לאחר מכן **Redeploy** (או פרסום אוטומטי בדחיפה הבאה).

### שלב 5 — בדיקה
שלח הזמנת בדיקה למספר שלך (שלא ביצע join). אמור להתקבל WhatsApp מהמספר המאושר,
בלי שום הצטרפות. אם יש שגיאה — נראה אותה ב-Twilio Console → Monitor → Logs → Messaging.

## עלות וזמנים (הערכה)
- **Twilio:** דמי הודעה לפי מדינה (בישראל — סנט בודדים להודעה) + ייתכן דמי מספר.
- **Meta:** תמחור WhatsApp לפי "conversations"; ל-Utility יש מכסה חינמית נדיבה.
- **זמן הקמה:** רוב הזמן הוא אימות העסק ב-Meta (שעות–ימים). הקוד מוכן מיידית.

## חלופה — Meta WhatsApp Cloud API ישירות
אם תעדיף לא דרך Twilio: `WHATSAPP_PHONE_NUMBER_ID` + `WHATSAPP_ACCESS_TOKEN`
(הקוד כבר תומך). גם שם נדרשים אימות עסקי + תבנית מאושרת; ל-Twilio יש UI נוח יותר
לתבניות, ולכן הוא הדרך המומלצת מכיוון שכבר מחוברים אליו.
